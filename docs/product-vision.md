# Product Vision

Status: durable reference

## Product

This project is a browser-first procedural building generator for stylized game environments.

The goal is not photorealism and not a full city simulator. The goal is a modular generation stack that can:

- generate plots and footprints
- extrude building massing
- decompose facades into semantic structure
- place architectural elements such as doors, windows, balconies, and trims
- assemble multiple buildings into a coherent small urban scene
- run end-to-end in the browser

The long-term aim is portability. The browser is the primary product surface, but the core generation logic should stay portable enough to move into another engine later without redesigning the whole system.

## North Star

The near-term target is a compact stylized street scene, not a single isolated building viewer.

The scene should include:

- a simple island-like or otherwise bounded presentation environment
- one main street with a few side branches
- roughly 20 buildings
- visible variation without feeling random
- enough architectural plausibility to read as one place

The result should feel lightweight, coherent, and clearly generated by reusable rules.

## Product Principles

### Browser first, engine portable

The browser implementation is the main working product. Core generation should still emit engine-neutral data contracts rather than browser-only logic.

### Semantic before geometric

The system should think in terms such as `plot`, `facade`, `bay`, `floor`, `window`, and `door`, not in terms of raw mesh files or renderer-specific scene structure.

### Rules over one-offs

Rules should adapt to varying widths and heights. A good facade rule describes a family of facades, not one exact wall.

### Data-first generation

Every important stage should produce structured outputs that can be inspected, tested, serialized, and reused.

### AI-agent-friendly development

The system should be easy for coding agents to change safely. That means deterministic inputs, inspectable intermediate outputs, explicit boundaries, and verification that does not rely only on human visual judgment.

## Near-Term Slice

The first meaningful slice should prove the full loop:

- seeded orthogonal plots
- simple building extrusion
- one facade rule with floors, windows, and a ground-floor door
- a minimal semantic asset library
- placeholder meshes or primitive proxies
- multi-building browser visualization
- deterministic debug artifacts for verification

## Non-Goals For Now

- photorealistic rendering
- full interior generation
- ultra-detailed bespoke ornament work
- traffic, population, or broader city simulation
- arbitrary architectural styles all at once

## Success Criteria

The project is on track if it can demonstrate that:

- a seed produces deterministic results
- simple rules adapt plausibly across different building sizes
- semantic asset requests can map onto interchangeable kits
- buildings can be generated end-to-end in the browser
- intermediate outputs are inspectable and testable
- the core concepts remain portable to another runtime
- multiple generated buildings read as a coherent urban slice
